# 全文搜索 v1 设计（中考知识库）

## 背景与目标

现有“中考知识库”具备上传、分类、转换、预览、删除、清空等能力，但缺少在已入库资料中的检索能力。需要新增“全文搜索”功能，支持在知识库中按关键词进行包含匹配搜索，并能快速定位到命中文件以便预览。

目标：

- 支持按关键词搜索（包含匹配，大小写不敏感）
- 默认同时支持：
  - 文件名匹配（任意类型文件）
  - 文本内容匹配（仅 `.md/.txt/.csv`）
- 返回命中片段，供前端展示
- 前端提供搜索框与结果列表，点击可直接打开预览

非目标（v1 不做）：

- 正则搜索
- 复杂分词/语义检索/向量检索
- 大规模索引系统（可在 v2 做增量索引）
- 对二进制内容（PDF 原件、图片原件）直接做全文检索（通过转换后的 `.md` 覆盖）

## 约束

- 前端部署在 Cloudflare Pages（纯静态）
- 后端部署在 Hugging Face Spaces（FastAPI）
- HF Space 可能休眠/重启；v1 不依赖长驻内存状态
- 为避免性能与带宽问题，搜索在后端执行，前端不拉取整库全文

## 用户体验（前端）

页面新增“全文搜索”区块：

- 输入框：关键词（必填，1–60 字符）
- 可选：学科筛选（默认“全部”）
- 按钮：搜索
- 结果列表：
  - 显示：学科、文件名、命中位置（filename/content）、片段（snippet）
  - 点击“预览”：复用现有预览弹窗，加载对应文件内容

## API 设计（后端）

### `GET /api/search`

Query 参数：

- `q`（string，必填）：关键词
- `category`（string，可选）：学科目录名（如 `语文`）
- `limit`（int，可选，默认 20）：最多返回多少个文件命中
- `context`（int，可选，默认 60）：命中片段前后截断长度（字符数）

返回 JSON：

```json
{
  "query": "文言文",
  "category": "语文",
  "limit": 20,
  "context": 60,
  "results": [
    {
      "category": "语文",
      "filename": "语文.md",
      "path": "knowledge_base/语文/语文.md",
      "count": 2,
      "matches": [
        { "where": "content", "snippet": "…文言文…", "pos": 1234 },
        { "where": "filename", "snippet": "语文.md", "pos": 0 }
      ]
    }
  ]
}
```

字段说明：

- `results[].count`：该文件匹配次数（filename 命中 + content 命中）
- `matches[].where`：`filename` 或 `content`
- `matches[].snippet`：用于 UI 展示的短片段
- `matches[].pos`：在原字符串中的大致位置（字符索引），v1 仅用于排序或后续扩展（不要求前端跳转定位）

### 搜索范围与规则（v1）

- 文件名匹配：
  - 遍历目标目录下文件（排除 `.meta.json`）
  - 文件名包含 `q`（大小写不敏感）则命中
- 内容匹配：
  - 仅扫描：`.md/.txt/.csv`
  - 内容包含 `q`（大小写不敏感）则命中
  - 每个文件最多返回 3 个 content 片段（避免响应过大）
- 排序：
  - `count` 降序
  - `filename` 升序（稳定）

### 性能与防护

- 参数校验：`q` 长度 1–60；`limit` 1–50；`context` 10–200
- 仅读取文本文件内容；对极大文件可做简单截断（例如最多读取前 N MB）以保护资源

## 前端数据流

1. 用户输入 `q` 并点击搜索
2. 前端 `fetch(API_BASE + "/api/search?q=...")`
3. 渲染结果列表
4. 用户点击某条结果的“预览”
5. 前端调用现有 `GET /api/file/{category}/{filename}` 并弹窗展示

## 迭代方向（v2）

- 增量索引：上传/删除时更新索引文件，提高搜索速度
- 关键词高亮与定位跳转
- 多关键词 AND/OR、简单分词
